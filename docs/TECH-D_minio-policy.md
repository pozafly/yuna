# TECH-D: MinIO 접근 정책 문서

> **문서 상태**: 1차 확정 (2026-02-27)

---

## 1. 스토리지 전략 요약

| 항목 | 결정 |
|------|------|
| 솔루션 | **MinIO** (S3 호환 오브젝트 스토리지) |
| 사용 범위 | 개발 환경 + 초기 운영 |
| 운영 확장 시 | AWS S3 또는 동등 서비스로 마이그레이션 가능 (S3 호환 API) |

---

## 2. 버킷 구성

| 버킷명 | 용도 | 접근 정책 |
|--------|------|----------|
| `originals` | 사진 원본 파일 저장 | **비공개** (public 금지) |
| `thumbs` | 썸네일 이미지 저장 (추후 리사이즈 파이프라인) | **비공개** (public 금지) |

> 버킷은 최소 2개. 추후 필요 시 `exports`(아카이브 다운로드용) 등 확장 가능.

---

## 3. 외부 공유 차단 정책 (핵심)

### 3-1. 기본 원칙

```
1. 모든 버킷은 public 접근 금지
2. MinIO URL을 직접 외부에 노출하지 않음
3. 이미지 접근은 반드시 BE(NestJS) 서버를 경유하여 권한 확인 후에만 허용
4. 다운로드 버튼 없음 (UX 정책)
5. 워터마크 없음, 원본 화질 유지 (정책 고정)
```

### 3-2. 이미지 접근 플로우

```
[FE: web]                [BE: api]                [MinIO]
    │                       │                        │
    │── GET /posts/:id ──►  │                        │
    │                       │── 권한 체크             │
    │                       │   (User, Baby, visibility)
    │                       │── presigned URL 생성 ──► │
    │                       │◄── presigned URL ────── │
    │◄── 응답 (presigned URLs 포함) ──────────────────  │
    │                       │                        │
    │── <img src="presigned URL"> ──────────────────► │
    │◄── 이미지 데이터 ──────────────────────────────── │
    │                       │                        │
    .  [presigned URL 만료 후 접근 시]                 .
    │── <img src="expired URL"> ────────────────────► │
    │◄── 403 Forbidden ─────────────────────────────── │
```

### 3-3. Presigned URL 정책

| 항목 | 값 | 설명 |
|------|---|------|
| 만료 시간 | **5분** (300초) | 짧은 만료로 링크 유출 시 피해 최소화 |
| 발급 조건 | 인증된 사용자 + Baby 접근 권한 + visibility 체크 | 서버에서 모든 체크 후 발급 |
| 갱신 방식 | FE가 필요 시 새 presigned URL 요청 | 자동 갱신 또는 on-demand |
| 지원 Operation | `GET` only | `PUT`은 업로드 전용 별도 엔드포인트 |

> **presigned URL은 단기 일회성 접근 링크이다.** URL 자체를 외부 공유해도 5분 후 만료되어 접근 불가.

## 4. 이미지 업로드 플로우

```
[FE: web]                [BE: api]                [MinIO]
    │                       │                        │
    │── POST /posts         │                        │
    │   (메타데이터 전송) ──►  │                        │
    │                       │── 권한 체크 (OWNER)     │
    │                       │── presigned PUT URL 생성 ──► │
    │◄── presigned PUT URL ──────────────────────────── │
    │                       │                        │
    │── PUT 원본 이미지 ────────────────────────────► │
    │◄── 200 OK ──────────────────────────────────── │
    │                       │                        │
    │── POST /posts/confirm │                        │
    │   (업로드 완료 확인) ──► │                        │
    │                       │── PostMedia 레코드 생성  │
    │◄── 201 Created ────── │                        │
```

> **업로드 방식**: Presigned PUT URL 방식 사용. 이미지가 FE에서 MinIO로 직접 업로드되어 BE 서버 부하를 줄인다.
> 업로드 완료 후 BE에 확인 요청을 보내 DB에 PostMedia 레코드를 생성한다.

## 5. 오브젝트 키 네이밍 규칙

```
originals/{babyId}/{postId}/{uuid}.{ext}
thumbs/{babyId}/{postId}/{uuid}.{ext}
```

| 세그먼트 | 설명 |
|---------|------|
| `{babyId}` | Baby 단위 격리 |
| `{postId}` | 게시물 단위 그룹핑 |
| `{uuid}` | 파일 고유 식별자 (충돌 방지) |
| `{ext}` | 원본 확장자 (jpg, png, heic 등) |

> **경로에 추측 불가능한 UUID를 포함**하여, 키 패턴을 알아도 다른 파일에 접근할 수 없게 한다.

## 6. 썸네일 생성 전략 (개요)

| 항목 | 결정 |
|------|------|
| 생성 시점 | 업로드 완료 후 **비동기** 처리 (큐 또는 이벤트 기반) |
| 생성 크기 | 추후 결정 (예: 400px, 800px) |
| 원본 보존 | 항상 `originals` 버킷에 원본 유지 |
| 화질 정책 | 원본 화질 유지 (압축 없음, 워터마크 없음) |

> 썸네일 생성 파이프라인 세부 사항은 구현 단계에서 확정한다.

## 7. Baby 삭제 시 스토리지 처리

| 시점 | 행동 |
|------|------|
| Baby 삭제 직후 | `originals/{babyId}/`, `thumbs/{babyId}/` 경로 접근 차단 (presigned URL 발급 거부) |
| 삭제 후 30일 이내 | 다운로드 전용 API로 아카이브 패키지 생성 (OWNER만 접근 가능) |
| 삭제 후 31일 이후 | `originals/{babyId}/`, `thumbs/{babyId}/` 실제 삭제 (스토리지 정리) |

## 8. 보안 체크리스트

| 항목 | 상태 |
|------|------|
| 버킷 public 접근 | ❌ 금지 (anonymous set none) |
| 직접 MinIO URL 노출 | ❌ 금지 (BE 프록시 또는 presigned URL) |
| presigned URL 만료 | ✅ 5분 (짧은 TTL) |
| 업로드 권한 체크 | ✅ OWNER 역할 확인 후 presigned PUT 발급 |
| 다운로드 권한 체크 | ✅ User + Baby + visibility 전체 확인 |
| 다운로드 버튼 | ❌ 미제공 (UX 정책) |
| 워터마크 | ❌ 없음 |
| 화질 압축 | ❌ 없음 (원본 유지) |
