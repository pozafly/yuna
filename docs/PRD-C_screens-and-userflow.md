## 2. 유저 플로우

### 플로우 A: 초대 → 가입 → 피드

```
[OWNER] 초대 발송 (S-OWN-04)
  └─► 이메일 발송 (초대 토큰 포함, 유효시간 30분)
      └─► [INVITED] 이메일 수신 → 링크 클릭
          └─► 초대 수락 화면 (S-AUTH-02)
              ├─► [신규 사용자] 회원가입 (S-AUTH-03) → 로그인
              └─► [기존 사용자] 로그인 (S-AUTH-01)
                  └─► 메인 피드 진입 (S-FEED-01)
                      └─► 초대된 Baby의 INVITED 게시물 열람
```

```
[토큰 만료/취소 케이스]
이메일 링크 클릭
  └─► 초대 수락 화면 (S-AUTH-02)
      └─► "초대가 만료되었습니다 / 취소되었습니다" 안내
          └─► OWNER에게 재초대 요청 (앱 내 안내 메시지)
```

---

### 플로우 B: OWNER — Baby 생성 온보딩

```
최초 회원가입 완료
  └─► Baby 생성 화면 (S-OWN-01) [필수 온보딩]
      └─► 이름·성별 입력 → Baby 생성 완료
          └─► 메인 피드 (S-FEED-01) [빈 피드 상태]
              └─► 첫 게시물 작성 (S-FEED-03) 또는 초대 발송 (S-OWN-04)
```

```
[둘째 이후 Baby 추가]
Baby 설정 화면 (S-OWN-05)
  └─► "Baby 추가" 탭
      └─► Baby 생성 화면 (S-OWN-02)
          └─► 새 Baby 생성 완료 → Baby 전환 가능
```

---

### 플로우 C: 게시물 작성 → 댓글

```
[OWNER] 메인 피드 (S-FEED-01)
  └─► 게시물 작성 버튼 → 작성 화면 (S-FEED-03)
      └─► 사진 업로드 (다중) → 내용 입력 → visibility 선택 (PRIVATE/INVITED)
          └─► 게시 완료
              ├─► [visibility=INVITED] 구성원에게 알림 발송 (앱 내 + 이메일 + 푸시)
              └─► [visibility=PRIVATE] 알림 없음

[INVITED or OWNER] 게시물 상세 (S-FEED-02)
  └─► 댓글 작성
      └─► 게시물 작성자(+ 기존 댓글 작성자)에게 댓글 알림 발송
```

---

### 플로우 D: Letter(편지) 작성

```
[OWNER or INVITED] Letter 목록 (S-LTR-01)
  └─► 편지 작성 버튼 → 작성 화면 (S-LTR-03)
      └─► 제목·본문 입력 → visibility 선택 (PRIVATE/INVITED)
          └─► 저장 완료
              ├─► [visibility=INVITED] OWNER에게 새 편지 알림 발송 (앱/이메일/푸시)
              └─► [visibility=PRIVATE] 본인만 열람 가능

[OWNER or INVITED] Letter 상세 (S-LTR-02)
  └─► 댓글 작성 가능 (접근 권한자에 한함)
```

---

### 플로우 E: 차단

```
[OWNER] 구성원 관리 (S-OWN-03)
  └─► 특정 구성원 선택 → "차단" 실행
      └─► 즉시 적용
          ├─► 해당 사용자: 모든 콘텐츠 접근 불가
          └─► 기존 콘텐츠(댓글/편지):
              ├─► 댓글: "차단된 사용자의 댓글입니다." 로 UI 표시
              └─► 편지: "차단된 사용자의 편지입니다." 로 UI 표시
```

---

### 플로우 F: INVITED 탈퇴

```
[INVITED] 개인 설정 (S-SET-01)
  └─► 탈퇴 요청
      └─► 탈퇴 확인 (경고: "탈퇴 시 접근 불가, 데이터는 유지됨")
          └─► 탈퇴 완료
              ├─► 해당 Baby 접근 완전 불가 (열람·작성 모두)
              ├─► 기존 데이터(사진·댓글·편지)는 서버에 유지
              └─► 탈퇴 당사자에게는 모든 콘텐츠 비노출
```

---

### 플로우 G: Baby 삭제 (OWNER)

```
[OWNER] Baby 설정 (S-OWN-05)
  └─► Baby 삭제 선택
      └─► Baby 삭제 확인 화면 (S-OWN-06) [필수 단계]
          ├─► 표시: "삭제 후 복구가 불가능합니다."
          ├─► 표시: "30일 이내 데이터 다운로드가 제공됩니다."
          ├─► 확인 입력 (예: Baby 이름 재입력) 후 삭제 실행
          └─► Baby 삭제 완료
              ├─► Baby + 모든 콘텐츠 접근 불가 (모든 구성원)
              ├─► OWNER 한한 OWNER만 다운로드 페이지 (S-OWN-07) 접근 가능
              └─► 30일 후 백엔드에서 데이터 완전 삭제
```

```
[Baby 데이터 다운로드 페이지 (S-OWN-07)]
다운로드 페이지 접근
  └─► 남은 다운로드 기간 표시 (XX일 남음)
      └─► 다운로드 시작 버튼
          └─► 실제 다운로드 (사진 원본파일 + 편지 텍스트 포함 패키지)
```

---

### 플로우 H: OWNER 탈퇴

```
[OWNER (1명)] 개인 설정 (S-SET-01)
  └─► 탈퇴 요청
      └─► 탈퇴 확인
          ├─► [공동 OWNER 존재 시] 경고: "탈퇴 후 접근 불가. 나머지 OWNER가 Baby를 지속 관리한다."
          └─► [마지막 OWNER 시] 경고: "탈퇴 후 Baby 데이터에 열람할 수 없음. (30일 다운로드 파인 안내)"
              └─► 탈퇴 전 Baby 삭제 또는 다운로드 제안
          └─► 탈퇴 완료
              ├─► 해당 OWNER Baby 접근 완전 불가
              └─► 데이터 서버에 유지 (기존 OWNER가 있으면 운영 지속)
```
