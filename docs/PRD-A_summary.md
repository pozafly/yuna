# PRD-A: 1페이지 요약 PRD

> **문서 상태**: 2차 업데이트 (2026-02-26)
> **서비스명(가칭)**: Yuna

---

## 1. 제품 목적

| 목적 | 설명 |
|------|------|
| 가족 공유 | 아기의 일상을 가족 구성원끼리만 공유 |
| 성장 아카이브 | 월별·마일스톤 기반으로 성장 기록을 체계적으로 보존 |
| 타임캡슐 | 아이에게 전하는 편지를 기록, 나중에 아이가 직접 열람 |
| 가족 커뮤니티 | 댓글·편지를 통한 가족 내 소통 공간 제공 |

---

## 2. 핵심 원칙 (절대 위반 금지)

1. **폐쇄형 서비스** — OWNER가 이메일로 초대한 사람만 접근 가능
2. **외부 공유 차단** — 인증·권한 없이 링크만으로 콘텐츠·이미지에 직접 접근 불가
3. **서버 최종 권한 판단** — 프론트엔드 숨김은 보안 수단이 아님; 모든 접근 제어는 서버에서 수행
4. **다운로드 버튼 미제공** — 사진 다운로드 기능 노출 없음
5. **원본 화질 유지** — 워터마크 없음, 화질 압축 없음 (화질 정책 고도화는 추후)

---

## 3. 사용자 및 역할

| 역할 | 정의 |
|------|------|
| **OWNER** | Baby를 생성·관리하는 사람. **공동 OWNER 허용** (예: 엄마/아빠 모두 OWNER 가능). 전체 관리 권한 보유 |
| **INVITED** | OWNER로부터 이메일 초대를 받고 승인한 가족 구성원 |

> Viewer 역할 없음. INVITED가 최소 접근 권한 단위.
> 공동 OWNER는 동등한 관리 권한을 가진다. OWNER 간 권한 차등은 이번 단계에서 없음.

---

## 4. 핵심 기능 리스트

### 4-1. Baby(아기) 관리
- OWNER가 Baby를 생성 (이름, 성별 입력)
- 둘째 이후 Baby를 추가 생성하여 구분 (**멀티 베이비** 지원)
- 초대는 Baby 단위로 관리 — INVITED는 초대된 Baby의 콘텐츠만 접근 가능
- **공동 OWNER**: OWNER가 다른 사람을 OWNER로 초대 가능 (동등한 관리 권한 부여)
- **Baby 삭제**: OWNER만 가능. 삭제 시 UI에서 **"삭제 후 복구 불가능"** 명시 필수
  - 단, 삭제 후 **30일간** 해당 Baby의 데이터(사진·편지·댓글) 다운로드 가능
  - 30일 경과 후 데이터 및 다운로드 접근 완전 불가

### 4-2. 초대(Invitation)
- 이메일 기반 초대만 허용
- 초대 토큰 만료: **30분**
- 재초대(새 토큰 발급) / 초대 취소(토큰 무효화) / 차단 가능
- 차단 즉시 접근 불가 / 차단된 사용자 콘텐츠는 삭제하지 않고 "차단된 사용자의 [댓글/편지]입니다." 로 표시

### 4-3. Post(사진 피드)
- 최신순 피드, 한 게시물에 사진 여러 장
- 게시물 내 사진: 좌우 Carousel UI
- **Visibility**: `PRIVATE`(OWNER만) / `INVITED`(초대된 모두)

### 4-4. Letter(편지 / 타임캡슐)
- 아기에게 쓰는 글 (타임캡슐 성격)
- **Visibility**: `PRIVATE` / `INVITED`
- 댓글 허용
- 작성일 기준 정렬·조회
- 예약 공개(미래 날짜 공개) — **이번 단계 제외, 추후 확장**

### 4-5. Comment(댓글)
- 볼 수 있는 권한이 있는 사람에게 댓글 전부 노출 (댓글만 따로 숨김 없음)
- 수정 가능 / 삭제는 **작성자 본인만** 가능 (OWNER도 타인 댓글 삭제 불가)
- 상위 콘텐츠(Post/Letter) 접근 권한이 없으면 댓글도 열람 불가

### 4-6. 타임라인 / 아카이브 뷰
| 뷰 | 설명 |
|----|------|
| 단순 피드 (Default) | 최신순 |
| 월별 앨범 | YYYY-MM 단위 |
| 성장 기록 | 마일스톤 중심 (D+100, 6개월, 1년 등) |
| 날짜 필터 | 특정 날짜·기간으로 필터링 |

### 4-7. 알림
| 이벤트 | 대상 | 채널 |
|--------|------|------|
| 새 게시물 | 접근 권한 있는 모든 구성원 | 앱 내 + 이메일 + 푸시 |
| 새 댓글 | 해당 게시물/편지 관련자 (최소: 작성자) | 앱 내 + 이메일 + 푸시 |
| 새 편지 | OWNER | 앱 내 + 이메일 + 푸시 |
| 초대 | 초대 대상자 | 이메일(필수) + 앱 내(선택) |

---

## 5. 피드 노출 규칙 요약

```
INVITED 사용자 피드 노출 기준:
  - 초대된 Baby의 콘텐츠 중 visibility = INVITED → 전부 노출
  - visibility = PRIVATE → 절대 노출 안 함
```

---

## 6. 탈퇴 정책

| 대상 | 접근 권한 | 기존 데이터 |
|------|-----------|-------------|
| INVITED 탈퇴 | 완전 불가 (열람/작성 모두) | 삭제 않고 서버에 유지 (사진/댓글/편지) |
| OWNER 탈퇴 (공동 OWNER 중 1명) | 탈퇴한 OWNER는 접근 불가 | Baby·콘텐츠 데이터 유지, 나머지 OWNER가 운영 지속 |
| OWNER 탈퇴 (마지막 OWNER) | 접근 불가 | Baby 데이터 서버에 유지 (고아 상태, 정책 추후 결정) |

> 탈퇴 당사자에게 모든 콘텐츠는 보이지 않음.

> **Baby 삭제 정책**: 탈퇴와 별개로 OWNER가 명시적으로 Baby를 삭제할 수 있음. 삭제 후 30일간 데이터 다운로드 가능, 이후 완전 삭제.

| 항목 | 상태 |
|------|------|
| ~~OWNER 다중 허용~~ | ✅ **확정**: 공동 OWNER 허용 |
| ~~OWNER 탈퇴 시 Baby 데이터 처리~~ | ✅ **확정**: 데이터 유지, 마지막 OWNER 탈퇴 시 고아 상태 처리는 추후 결정 |
| 마지막 OWNER 탈퇴 시 Baby 고아 상태 처리 | **미정** |
| 장기 저장·백업·아카이브 내보내기 정책 (서비스 종료 등) | **미정** |
| Letter 예약 공개 (미래 날짜 공개) | 추후 확장 |
| 선택 공개 (특정 인원만) 기능 | 추후 확장 |
