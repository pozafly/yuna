# Knowledge Search (KI) 스킬

**목적:** 과거 회의록, 기획 변천사, 이전 사용자(OWNER)의 요구사항 등 누적된 컨텍스트를 빠르게 검색하고 요약하여 현재 작업(특히 기획 및 요구사항 정의)에 반영합니다.

**사용 시점:**
- 새로운 기능을 기획할 때 과거 논의 내용을 확인할 필요가 있을 때
- 기존 정책의 유래나 이유를 알아야 할 때
- 프로젝트 도메인(육아, 권한)에 대한 지식을 빠르게 파악해야 할 때

**사용 방법 (가이드):**
- 파일시스템 내 `.gemini/brain` 또는 유사한 지식 저장 디렉토리의 컨텍스트를 읽어옵니다.
- 검색된 결과가 방대할 경우 요약 기능과 병행하여 필요 핵심 정보만 추출합니다.
