# Database Client & SQL Execution 스킬

**목적:** 서버 로직 동작 확인을 위해 데이터베이스에 직접 질의(Query)하거나 변경사항을 반영하여 올바르게 CRUD 및 권한/트랜잭션이 작동했는지 저수준 검사를 수행합니다.

**사용 시점:**
- Baby, Letter, Post 엔티티에 대한 릴레이션이 깨지지 않았는지 데이터 무결성을 확인할 때
- API 호출 전후로 데이터베이스의 레코드 상태(저장, 삭제) 및 타임스탬프가 정상인지 대조할 때
- 마이그레이션 스크립트에 의해 스키마가 의도한 대로 갱신되었는지 조회할 때

**사용 방법 (가이드):**
- 직접적인 SQL 쿼리 실행 또는 ORM 전용 커맨드(CLI) 기능을 통해 트랜잭션을 실행합니다.
- 조회용 `SELECT` 이외의 작업 시행 시 `ROLLBACK` 가능한 환경인지 미리 파악하세요.
