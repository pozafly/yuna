# Browser Vision & DOM Interaction 스킬

**목적:** 코드로 구현된 UI가 실제로 어떻게 렌더링되는지 브라우저에서 캡처하여 확인하고, 디자인 가이드라인 준수 여부를 검수(Visual QA)합니다.

**사용 시점:**
- 프론트엔드 에이전트가 특정 화면 퍼블리싱을 완료했을 때
- CSS 여백, 정렬, 색상(명도 대비), 모서리 둥글기 수준이 올바른지 디자인 리뷰가 필요할 때
- 로컬 서버(`localhost:3000`)에 접속하여 버튼 클릭이나 모달 작동 등 인터랙션을 흉내내고자 할 때

**사용 방법 (가이드):**
- 브라우저 서브에이전트 도구를 호출하여 로컬 URL에 접속합니다.
- 렌더링된 화면 스크린샷 캡쳐나 DOM 요소 스니펫을 읽어 디자인 가이드라인(`docs/TECH-F_design-guidelines.md`)과 대조합니다.
- 지적 사항은 모호한 표현 대신 구체적인 CSS 속성 단위(예: `border-radius: 9999px`)로 피드백합니다.
