# Git Operations 스킬

**목적:** Yuna 프로젝트의 코드 변경 이력을 추적하고, 안전한 브랜치 전략을 통해 병합 충돌을 최소화하며, 코드 리뷰를 위한 변경 사항 요약을 제공합니다.

**사용 시점:**
- 새로운 기능/버그 수정을 위한 피처 브랜치를 생성·전환할 때
- 작업 전후의 변경 사항(diff)를 파악하여 영향 범위를 확인할 때
- 과거 커밋 이력에서 특정 변경의 원인이나 맥락(blame)을 추적할 때
- 브랜치 병합 시 충돌이 발생하여 수동 해결이 필요할 때

**사용 방법 (가이드):**
- `git diff --stat`으로 변경 파일 목록을 먼저 확인한 뒤, 개별 파일의 diff를 상세 열람합니다.
- 커밋 메시지는 `[영역] 변경 요약` 형태(예: `[shared-types] BabyDTO에 profileUrl 필드 추가`)로 작성하여 에이전트별 작업 영역을 명확히 합니다.
- 병합 전 항상 `pnpm type-check && pnpm lint`을 실행하여 타입/린트 오류가 없는지 확인합니다.
